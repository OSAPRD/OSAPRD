# Generated GitHub Topic Rule Profile

This directory is generated by:

```bash
python post-processing/topic-classification/data-preparation/generate_github_topic_rules.py
```

The generated profile keeps the same rules/list layout as
`data-preparation/rules` and `data-preparation/lists`, but derives topic aliases
and canonical topic mappings from the local GitHub topics catalog.

## Files

- `rules/*.csv`: generated runtime rule files consumed by
  `classify-topics/topic_preprocessing.py`.
- `lists/*.txt`: copied or generated dictionary files consumed by
  `classify-topics/topic_preprocessing.py`.

Treat these files as generated artifacts. Regenerate them from the source
catalog instead of hand-editing rows.

The generator also writes `github_topic_rules_manifest.json`,
`github_topic_catalog.json`, `github_topic_alias_map.csv`, and
`github_topic_legacy_rule_merge_audit.csv` when it runs. Those files are
audit/provenance outputs and are not required by the active training or
classification pipeline, so they are not kept in this tree.
